# Power Spectra Analysis Report

## Overview

This report documents the power spectrum analysis of true hit data from particle detector simulations. The plots analyze the frequency content of the reconstructed charge signals across three spatial axes (x, y) and the time axis (t).

## Data Source

**File Pattern:** `power_spectra_true_pgun_muplus_3gev_*.png`

- **Particle:** μ+ (muon)
- **Energy:** 3 GeV
- **Signal Type:** True hits (ground truth, noise-free)

The data is processed through:
1. Raw hit data → Interval-averaged charge blocks
2. Active region detection (10% threshold of max projection)
3. Per-axis power spectrum computation via FFT
4. 2D histogram accumulation of frequency vs. power

## Plot Structure

Each figure contains **three subplots** (one per analysis axis):

### Subplot Layout

| Axis | Analysis | Frequency Unit | Description |
|------|----------|----------------|-------------|
| **X (pixel)** | FFT along x-direction | cycles/pixel | Spatial modulation in detector x-direction |
| **Y (pixel)** | FFT along y-direction | cycles/pixel | Spatial modulation in detector y-direction |
| **T (tick)** | FFT along time-direction | cycles/adc_hold_delay | Temporal evolution within burst readout |

## Visualization Elements

### 2D Histogram (Background)
- **X-axis:** Frequency (cycles per unit)
- **Y-axis:** log₁₀(power) for all traces in the region
- **Color:** Viridis colormap (dark=few traces, bright=many traces)
- **Interpretation:** Shows the distribution of power across frequencies; clustering indicates dominant frequency components

### Mean Power Overlay (Red Line)
- **Overlay on histogram:** Average power at each frequency
- **Calculation:** Mean of log₁₀(power) across all traces
- **Interpretation:** The dominant frequency signature of the true signal
- **Peaks:** Indicate characteristic frequencies present in the data

## Interpretation Guide

### High Mean Power (Red Line Peaks)
- Frequencies where the true signal carries significant energy
- For **spatial axes:** Indicates preferred length scales or periodic structures in the detector response
- For **time axis:** Shows dominant frequencies in the burst structure or integration pattern

### Low/Flat Power Regions
- Frequencies with minimal contribution from the signal
- Useful for identifying frequency ranges dominated by noise (in noisy data comparison)

### Active Region Slicing
- Only the active signal region (10% threshold per axis) is analyzed
- Eliminates padding and dead space from distorting the spectrum
- Parameters applied: `regions = find_active_region(true_meas, threshold=0.10)`

## Comparison Context

When paired with `power_spectra_noisy_*.png`:
- **True spectra:** Show intrinsic signal frequency content
- **Noisy spectra:** Show signal + noise mixture
- **Ratio plots:** Quantify noise corruption (SNR in frequency domain)

## Technical Details

### FFT Configuration
- **Method:** `np.fft.rfft` (real-valued FFT, one-sided spectrum)
- **Windowing:** None (rectangular window)
- **Normalization:** Power = |FFT|²

### Histogram Parameters
- **Frequency bins:** 100
- **Power bins:** 100 (log-space)
- **Log scale:** Yes (log₁₀ applied to power values)

### Data Normalization
- Interval-averaged bursts with `adc_hold_delay` integration windows
- Charge values represent integrated current over fixed time intervals
- Time frequency unit = cycles/adc_hold_delay (typically ~0.05 μs)

## Notes for Interpretation

1. **Frequency Resolution:** Limited by FFT size (number of samples along each axis)
   - Larger spatial extent → better frequency resolution in space
   - Longer signal duration → better temporal frequency resolution

2. **DC Component (f=0):**
   - Appears in all plots as the baseline
   - Represents the total integrated charge

3. **Nyquist Frequency:**
   - Upper limit = 0.5 cycles/sample
   - For time: 0.5 / adc_hold_delay ~ 1/60 / tick (for 0.05 μs tick)

4. **Interpretation by Axis:**
   - **X/Y:** Diffusion patterns, pad geometry, field response
   - **T:** Drift time structure, shaping time constant, integration window effects
